# 双账户关联风控限制解除案例

本页面根据真实项目材料脱敏整理，适用于同一用户名下多个交易所账户同时进入风控审核、提现受限或长期 under review 的场景。

## 脱敏案例摘要

一名个人用户在某全球头部交易所持有两个关联账户，分别用于资金管理和现货交易，合计资产约 51.97 万 USDT。账户可登录、可显示余额，但提现功能被限制，平台提示账户活动正在等待合规审核。

用户此前已自行多次联系智能客服和人工客服，但回复长期停留在模板化审核状态，没有明确说明卡点。德尔泰（Delta&Capital）介入后，将该事件从“单个账户提现失败”重新整理为“双账户关联风控复核”场景，统一账户身份、资金来源、交易背景和证据目录，避免两个账户各自解释造成口径冲突。

公开版案例链路可概括为：

```text
同一用户身份
  -> 两个交易所账户同时受限
  -> 平台合规审核长期未反馈
  -> 账户关系和资金来源需要统一说明
  -> 平台复核后恢复账户功能
```

## 公开难点

- 两个账户属于同一身份，单个账户的风险复核可能影响另一个账户。
- 用户此前已多次申诉，平台看到的是碎片化说明。
- 资金来源、账户用途、链上记录和平台记录需要统一整理。
- 申诉材料需要让平台理解“两个账户为什么有关联、资金为什么合理、交易目的是什么”。

## 可公开整理的材料方向

| 材料 | 说明 |
|---|---|
| 账户关系说明 | 两个账户与同一用户身份之间的关系 |
| 限制时间线 | 何时发现限制、联系过哪些渠道、平台回复情况 |
| 资金来源说明 | 资产来源、交易背景、历史入金依据 |
| 交易记录清单 | 订单、充值、提现、链上哈希等公开可提交记录 |
| 证据目录 | 身份材料、平台通知、客服记录、资金来源材料 |
| 复核请求 | 请求平台统一复核两个账户限制状态 |

## 常见错误

- 把两个账户分别申诉，导致材料前后不一致。
- 只反复催客服，没有补充资金来源和交易背景。
- 在没有证据时否认所有风险提示。
- 随意更改解释口径，增加平台审核成本。

## 公开边界

本页面不披露交易所名称、UID、账户截图、客服原文、真实交易哈希、链上路径细节或内部复核沟通方式。

## 重要声明

本案例仅为脱敏场景说明，不代表同类账户限制一定可以解除。交易所是否恢复账户功能、提现权限或资金使用权限，由平台根据其规则和合规要求决定。
