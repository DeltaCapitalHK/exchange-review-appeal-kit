# 交易所账户风控申诉材料清单

## 目的

适用场景：币安账户被冻结、欧易 OKX 风控、Bybit 提现审核、Coinbase 账户 under review、Kraken 资金审核、Gate / KuCoin / Bitget 账户限制、HTX/老火币账号停用或历史资产找回、P2P/C2C 功能被限制、平台要求资金来源说明等。

本清单的目的不是保证解冻，而是帮助用户把事实和证据整理清楚，便于向平台客服、合规审核团队或专业顾问说明情况。

## 适用场景

当交易所账户功能被限制、提现进入审核、平台要求补充资金来源、C2C/P2P 功能受限或账户进入风控审核时，可以先按本清单整理材料。

## 1. 账户与限制信息

请先整理：

- 平台名称
- 账户 UID / 用户 ID
- 注册邮箱或手机号
- 被限制功能：登录、交易、充值、提现、P2P/C2C、API、法币、银行卡等
- 限制发生时间
- 平台提示原文截图
- 邮件、App 通知、网页弹窗或客服回复截图
- 工单编号、申诉编号、订单编号或 case ID

## 2. 事件时间线

按时间顺序整理，不要只写结论。

| 时间 | 事件 | 证明材料 |
|---|---|---|
| 2026-xx-xx xx:xx | 平台出现账户限制提示 | 截图 A01 |
| 2026-xx-xx xx:xx | 尝试提现失败或进入审核 | 截图 A02 / 交易记录 |
| 2026-xx-xx xx:xx | 提交客服工单 | 工单 B01 |
| 2026-xx-xx xx:xx | 平台要求补充资金来源 | 邮件 C01 |

## 3. 涉及资金与交易信息

请整理：

- 涉及币种：BTC、ETH、USDT、USDC、法币等
- 涉及金额
- 涉及时间范围
- 平台订单号
- 充值和提现记录
- 链上交易哈希
- 发送和接收钱包地址
- 交易对手信息，如可获得
- 资金来源类型：工资、经营收入、投资收益、OTC/P2P、其他交易所转入、钱包转账、客户/朋友/合作方付款等

## 4. 资金来源证明

根据实际情况准备：

- 银行流水
- 工资或经营收入证明
- 合同、发票、付款说明
- OTC/P2P 订单截图
- 交易所订单历史
- 钱包交易历史
- 税务、会计或业务记录
- 与交易对手的沟通记录
- 区块浏览器截图

## 5. 说明材料怎么写

有效说明应该：

- 基于事实
- 按时间顺序
- 与截图、订单、交易哈希一致
- 只说明相关事实
- 不夸大、不保证、不编造
- 对无法确认的信息明确说明“暂无法确认”

避免：

- 多次提交互相矛盾的说法
- 一次性上传大量无关截图
- 没有证据就说“资金绝对合法”
- 只催促客服但不补充材料
- 相信所谓“保证解冻”“内部通道”“绕过风控”的说法

## 6. 提交后的记录

提交后请保留：

- 提交时间
- 工单编号或申诉编号
- 提交的附件清单
- 平台每次回复
- 后续补充材料记录

除非平台要求，不建议反复创建多个内容矛盾的工单。
